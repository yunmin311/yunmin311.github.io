# 迁移备忘 — 搬去别的托管平台(人话版)

> 和 `搬家清单.md` 分工:那份管**换电脑**,这份管**换托管平台**
> (不想用 GitHub Pages 了,搬去 Netlify / Cloudflare Pages / 自己的服务器)。

## 先放心:这个站没有被任何平台绑死

网站是**纯静态**的——构建之后就是一个装满普通网页文件的 `dist` 文件夹,
任何能放静态文件的地方都能托管它,不需要数据库、不需要后端程序。

全仓库只有 **两处** 写着 GitHub Pages 的网址,迁移时各改一行:

| 在哪 | 改什么 |
|------|--------|
| `astro.config.mjs` 顶部的 `SITE` 常量 | 换成新网址 |
| `public/robots.txt` 的 `Sitemap:` 行 | 网址部分同步换 |

(`.github/workflows/deploy.yml` 是 GitHub Pages 专用的自动部署,搬走后删掉或留着不用都行。)

## 通用参数(三条路都一样)

- 构建命令:`npm run build`
- 发布目录:`dist`
- Node 版本:22 或更新

## 路线一:Netlify

1. 注册 netlify.com,选 "Import from Git",授权后选中这个仓库;
2. 构建命令和发布目录填上面的通用参数,点部署;
3. 拿到 `xxx.netlify.app` 网址(或绑自己的域名)后,改上面那两处网址,提交推送。
   以后每次推送 GitHub,Netlify 自动重新部署。

## 路线二:Cloudflare Pages

1. 注册 cloudflare.com → Workers & Pages → 创建 → 连接 Git,选中仓库;
2. 框架预设选 Astro(参数会自动填对),点部署;
3. 拿到 `xxx.pages.dev` 网址(或绑域名)后,同样改两处网址,提交推送。

## 路线三:自建服务器

1. 在任何一台电脑上跑 `npm run build`,得到 `dist` 文件夹;
2. 把 `dist` 里的**全部内容**上传到服务器网站根目录(nginx、Apache、宝塔面板都行,
   就是普通静态文件,不需要装 Node);
3. 改两处网址,以后每次更新内容都重新构建、重新上传一次 `dist`。

## 域名说明

绑自己的域名(如 `example.com`)属于各平台的常规操作,三家都有图形界面引导;
绑完之后记得回来改那两处网址。

# 个人网站 — 功能规格(spec)

日期:2026-07-16(同日经十轮澄清后更新)。上游文档:`2026-07-16-personal-website-design.md`
(设计方案,用户已批)、`constitution.md`(项目宪法)。
本规格把设计方案细化到"可以照着写代码"的程度,所有决策均经用户逐条确认。

## 一、定位(不变,来自设计方案)

设计师的个人网站:作品集为主、博客为辅,中英双语,GitHub Pages 免费托管。
访客 10 秒内知道站主是干什么的。

## 二、澄清环节定下的决策(2026-07-16,共十一轮)

### 基础
| 问题 | 决定 |
|------|------|
| 默认语言 | **中文**,根网址自动跳 `/zh/` |
| 内容只有单语版本时 | 另一语言站**直接隐藏**,不展示不提示 |
| 站名 | **未决**,先用占位符,集中定义、将来一处替换 |
| 访问统计 | **不要** |

### 首页
| 问题 | 决定 |
|------|------|
| 自我介绍与精选作品的关系 | **介绍块嵌在网格里**:介绍文字作为网格中的一格,与作品图块平起平坐(杂志拼贴感,Apartamento 一路) |
| 精选作品展示 | 网格(两三列),`featured: true` 按 `order` 排序 |
| 页头导航 | **不随滚动常驻**,随页面滚走 |

### 作品集
| 问题 | 决定 |
|------|------|
| 列表页形态 | 与首页**同款网格**,不嵌介绍块 |
| 作品分类 | **先不分**,字段预留,将来要分随时加 |
| 预期体量 | 初期 5–15 个,**不做分页** |
| 详情页图片排布 | **单列大图长滚**,文字穿插其间 |
| 作品字段 | 年份(用 date)、**客户/项目归属、担任角色、外部链接**全要,选填、不填不显示 |
| 详情页翻页 | **要**"上一个/下一个作品"入口 |
| 封面比例 | **统一裁成固定比例**(具体比例视觉定稿时定),任意图自动居中裁切 |
| 视频 | 预留视频位,存 Vimeo/YouTube/B站 链接,不存视频文件 |

### 博客
| 问题 | 决定 |
|------|------|
| 内容类型 | 设计随笔、项目过程记录、生活内容**都会写** |
| 列表页 | **纯文字清单**:标题 + 日期 + 一句简介 |
| 标签/分类、RSS | 都不要,先最简 |
| 阅读时长、目录 | 都不要 |

### 关于/联系
| 问题 | 决定 |
|------|------|
| 照片 | **要,留照片位**(照片后补) |
| 经历/履历区 | 不要,详细经历放简历 PDF |
| 目前状态 | **要,留一行状态位**(如"现居某地,接受合作",内容随时改) |
| 联系方式 | 邮箱(防爬处理)+ 社交/平台链接 + 简历 PDF 下载 |
| 简历 | **中英各一份**。⚠️ 仓库公开、PDF 公开可下载,**上线前必须抹掉电话、住址等隐私信息** |

### 语言与文字
| 问题 | 决定 |
|------|------|
| 导航用词 | 先用通用版:中文"作品 / 博客 / 关于",英文 "Works / Blog / About",定稿时再改(集中在字典文件,改一处) |
| 日期格式 | 随语言各自习惯:中文站"2026 年 7 月",英文站 "July 2026" |
| 切换控件 | **EN / 中 两词并列**,当前语言加深或下划线 |
| 记住语言选择 | **不记**,每次从根网址进都到中文 |

### 页面骨架
| 问题 | 决定 |
|------|------|
| 内容宽度 | **宽幅面**:网格和大图铺开用足大屏,正文段落限制在舒适阅读宽度 |
| 动效 | **几乎静止**:只有链接悬停变色等基本反馈,无滚动动画 |
| 手机导航 | **直接排一行小字**,不做汉堡菜单 |

### 内容供给工作流
| 问题 | 决定 |
|------|------|
| 供稿方式 | 用户在聊天里发文字+图,由对话排成标准格式,用户预览确认后提交 |
| 图片格式 | JPG/PNG 导出图 + 手机照片(HEIC/大图,对话转格式压缩);构建自动出各屏幕版本 |
| 视频 | 视频文件不进仓库,用户传到 Vimeo/B站 等平台,网站只嵌链接 |
| 网址短名 | 对话提议英文短名、用户否决制;上线后网址不轻易改 |

### 搜索与分享
| 问题 | 决定 |
|------|------|
| 搜索引擎收录 | **要**,配 sitemap 等基础设施 |
| 分享卡片图 | 作品页用该作品封面;首页/博客等用全站统一图(占位图先顶,视觉定稿后换) |
| 站点一句话描述 | 对话先写中英占位各一句,集中存放,用户随时改 |

### 上线与运维
| 问题 | 决定 |
|------|------|
| 上线时机 | **骨架就绪就上**(占位内容也先上,尽早验证部署链路) |
| 网站图标(favicon) | 占位期用**站名首字母**的极简文字图标(现 Astro 火箭默认图标必须换掉) |
| 更新验收 | **每次改动本地预览、用户确认后才推送上线** |

## 三、全站通用

- **页头**:站名(占位)+ 导航(作品 / 博客 / 关于)+ 语言切换(EN / 中 并列);
  **不随滚动常驻**;手机上同样一行排开,不做汉堡菜单
- **页脚**:© 年份 + 站名占位 + 社交链接(与关于页共用同一份数据)
- **语言切换规则**:中英内容同名配对;当前页有对应翻译就跳对应页,
  没有(含"该内容另一语言无版本")就跳该语言的首页;不记住访客的语言选择
- **草稿**(`draft: true`)构建时排除,线上完全不可见
- **404 页**:双语提示 + 回首页链接
- **响应式**:手机、平板、桌面都可正常阅读;宽幅面取向(图铺开、正文限阅读宽度)
- **动效**:几乎静止,仅基本悬停反馈
- **基本 SEO**:每页标题与描述;og 分享卡片(作品页用封面、其余统一图);
  hreflang 中英互指;sitemap;允许搜索引擎收录
- **无 cookie、无统计、无任何第三方脚本**

## 四、页面规格

### 首页 `/zh/`、`/en/`
- 网格(两三列):**介绍块作为网格中的一格**(我是谁、做什么,几行字),
  其余格子是精选作品(`featured: true` 按 `order` 排,内容规划上建议 3–6 个)
- 入口:查看全部作品、博客、关于

### 作品集列表 `/zh/works/`、`/en/works/`
- 与首页同款网格,不嵌介绍块;全部非草稿作品按 `date` 倒序;不分页
- 每项:封面图(统一比例裁切)+ 标题 + 一句简介

### 作品详情 `/zh/works/<短名>/`、`/en/works/<短名>/`
- 头部:标题 + 年份 + 简介;客户/角色/外链有则显示
- 正文:markdown 自由排版,图片单列大图长滚,文字穿插
- 视频位:frontmatter 给链接则渲染内嵌播放器,不给则无此区域
- 尾部:上一个/下一个作品翻页入口

### 博客列表 `/zh/blog/`、`/en/blog/`
- 纯文字清单:标题 + 日期 + 一句简介,非草稿按 `date` 倒序

### 文章页 `/zh/blog/<短名>/`
- 标题、日期、正文(markdown);无阅读时长、无目录

### 关于/联系 `/zh/about/`、`/en/about/`
- 照片位(照片后补)+ 自我介绍(markdown)
- 一行"目前状态"位(内容随时改)
- 邮箱(防爬:不以明文出现在网页源码)+ 社交/平台链接
- 简历 PDF 下载,中英各一份。**上线前提醒用户抹隐私信息**

## 五、内容模型(markdown 文件的信息头格式)

目录结构:同一内容的中英版**文件名相同**,放各自语言文件夹:

```
src/content/works/zh/poster-series.md
src/content/works/en/poster-series.md   ← 同名 = 互为翻译
src/content/posts/zh/hello.md
src/content/posts/en/hello.md
```

**作品(works)**:

| 字段 | 必填 | 说明 |
|------|------|------|
| title | ✓ | 标题 |
| summary | ✓ | 一句话简介(列表、分享卡片用) |
| date | ✓ | 日期(排序、年份显示用) |
| cover | ✓ | 封面图(统一比例裁切展示) |
| client | | 客户/项目归属 |
| role | | 担任角色 |
| link | | 项目线上地址 |
| video | | 视频链接(Vimeo/YouTube/B站) |
| featured | | 是否精选,默认否 |
| order | | 精选区排序,小数在前,默认 0 |
| draft | | 草稿,默认否 |

**文章(posts)**:

| 字段 | 必填 | 说明 |
|------|------|------|
| title | ✓ | 标题 |
| summary | ✓ | 一句话简介 |
| date | ✓ | 日期 |
| draft | | 草稿,默认否 |

图片文件与内容文件放在一起,构建自动优化压缩。

## 六、明确不做(继承设计方案 + 澄清补充)

CMS、评论、账号、会员、前端框架、博客标签/分类、RSS、访问统计、站内搜索、
深浅色切换、作品分类筛选(字段预留)、分页、汉堡菜单、滚动动画、记住语言偏好。

## 七、未决事项

| 事项 | 状态 | 影响 |
|------|------|------|
| 站名 | 用户说"后面再说" | 占位符集中定义,替换只改一行;favicon 首字母随之 |
| 视觉方向 | 设计辅助对话选方向中 | 占位样式,变量集中 `tokens.css`,定稿只改一处 |
| 封面统一比例的具体值 | 视觉定稿时定 | 先用 4:3 占位 |
| 导航用词定稿 | 视觉/文案定稿时 | 字典文件一处改 |
| 真实内容 | 待用户供稿 | 示例占位内容,明确标注 |
| 简历 PDF、关于页照片 | 待用户提供 | 页面留位 |
| 分享统一图 | 视觉定稿后换 | 先用占位图 |

## 八、整体验收标准

1. `npm run build` 通过,产物纯静态
2. 根网址自动跳中文首页;每页可一键切换语言;规则符合第三节
3. 只有单语版本的内容,在另一语言站完全不可见
4. 草稿内容线上完全不可见
5. 手机上各页可正常阅读操作(导航一行排开)
6. 推送到 main 后网站自动更新(部署阶段起)

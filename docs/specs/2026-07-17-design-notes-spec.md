# 设计整合板块 — 需求文档(2026-07-17 用户澄清确认)

> 九大板块之六,独立型的最后一个。三轮需求澄清的落字结果,**只定义、不搭建**:
> 页面代码要等视觉方向定稿(`tokens.css`)且用户点名开工才写(宪法第八条,先规格后代码)。
> 体例参照 `2026-07-17-photo-gallery-spec.md`。
> 涉及用户另一个私有工具项目时,本文档一律脱敏、不写项目名(本仓库公开)。

## 一、定位:它是什么、和素材库的边界在哪

愿景里的一句话:设计爬取、整理后的**可复用设计沉淀**。定位是双重的:
先是**自己的沉淀**(公开的个人设计笔记本,自己随时翻),顺带**对外分享**
(设计同行看得懂、用得上)。

**和素材库的边界**(两者都涉及设计物,用户确认的切法):

- 素材库放「**可复用的物件**」——拿来直接用的文件/信息卡;
- 设计整合放「**可复用的认知**」——规则、数值、方法、案例,读了改变你怎么做设计。

同一批研究既产物件又产认知,就**拆开各归各位**,不迁就放一起。

## 二、基本盘(已拍板)

- **内容四类**(还会有更多,留活口):
  1. 蒸馏出的规则/方法(自制文字,版权干净);
  2. 实测数值/真值(从优秀网站实测扣出的字号、间距、配色等硬数据整理);
  3. 风格研究报告(见下:重新整理后进站,不挂原件);
  4. 自己的落地案例(研究用到自己设计里的前后对照)。
- **报告进站形态**:现有研究产出多是双击即看的 HTML 报告文件——进站时
  **重新整理成网站文章**(md),不挂原 HTML。每篇多一道整理工序,
  换来全站视觉统一,且整理过程正好把截图版权关过掉(见第四节)。
- **管理地**:整理后的文章直接在网站仓库管(md 文件),不经过 Obsidian。
- **双语(已拍板):像作品集那样可选**——写了英文版就双语互切,没写就只在
  中文站出现。这是字最多的板块,不让翻译工作量拖着不敢发。
- **导航名字**:中文站「**设计整合**」,英文站「**Design Notes**」。

## 三、内容模型:一条 = 一篇文章(md 文件)

字段(用户确认真会填的):

| 字段 | 说明 |
|------|------|
| 标题 + 日期 | 最低配置 |
| 一句话简介 | 列表里告诉读者这篇整理了什么 |
| 主题分类 | 排版/配色/字体/交互…(可扩),列表按主题归组 |
| 类型标记 | 规则蒸馏 / 实测数值 / 研究报告 / 落地案例,四选一 |

正文是普通 markdown 文章;自己的配图过图片管线(`docs/图片入库工序.md`)。

## 四、内容从哪来 + 截图红线(本板块最重要的一节)

**来源(脱敏说法)**:源头产出在设计辅助对话与设计库那边;凡涉及用户另一个
私有工具项目的研究成果,照宪法经设计库 `public\` 层流入,本站不写该项目
的名字与细节。进站的最后一步永远是"重新整理成本站文章"——
**人工整理不是负担,是版权过滤器**。

**截图红线**(用户选择"想带少量截图",代价如下):

- 公开网站放他人网站截图 = 向全网再分发。**注明出处不等于合法**;
- 想留的截图**每张单独过版权关**,由用户逐张拍板;拿不准的不上;
- 原报告里成批的参考截图,重新整理时**默认剥离**,用文字描述 + 实测数值替代
  (数值是事实,版权干净);
- 设计库 `private\` 层的材料(他人截图、付费/禁止再分发物)永远不直接进站。

## 五、页面结构

| 页面 | 内容 | 定稿程度 |
|------|------|----------|
| 设计整合列表 | 按主题归组的文章列表,类型标记可见 | 只定底线 |
| 文章页 | 普通文章版式,和全站视觉统一 | 结构已定 |

具体版式交给视觉设计;这里只定"列表按主题能归组、类型一眼可辨"的底线。

## 六、入库工作流(发一篇的完整动作)

1. 选一篇源头产出(研究报告/蒸馏文字/数值整理/案例);
2. **重新整理成 md 文章**:剥离他人截图(想留的逐张报用户过版权关),
   自己的配图过 `scripts/prep-image.mjs`;
3. 填字段:标题、日期、一句话简介、主题分类、类型标记;英文版可选;
4. 本地预览确认 → `npm run check` + `npm run build` 双关 → 提交。

## 七、明确不做(YAGNI)

- 不把原 HTML 报告文件挂进站(见第二节);
- 不做设计库→网站的自动同步(人工整理是特性,见第四节);
- 不做搜索、评论;主题分类够用,不做标签系统;
- 不做 Obsidian 对接(整理后的文章直接在仓库,理由同前几个板块)。

## 八、开工条件

同时满足才动手:① 视觉方向定稿(`tokens.css` 就位);② 用户点名开工。
它有现成的源头存货(风格研究两辑、蒸馏军规等),开工时从存货里选头几篇,
届时实现细节另出实现计划,仍先给用户过目。

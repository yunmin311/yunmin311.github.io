# 学习记录板块 — 需求文档(2026-07-18 用户澄清确认)

> 九大板块之八,三个文字类板块的第二个。两轮需求澄清的落字结果,**只定义、不搭建**:
> 页面代码要等视觉方向定稿(`tokens.css`)且用户点名开工才写(宪法第八条,先规格后代码)。
> 体例参照 `2026-07-17-photo-gallery-spec.md`。
> **对接机制不在本文档里**:怎么从 Obsidian 进站已在 `2026-07-18-obsidian-sync-spec.md`
> 定死,本文档只定这个板块**自己**的形状,不重开地基。

## 一、定位:它是全站边界最多的板块,三刀都已切好

学习记录 = **学的轨迹**,写给自己、顺带给人看,笔记可以粗糙、片段、半成品。
它和三个近邻的分家(全部用户拍板):

1. **vs 博客 —— 过程 vs 成文**:这里放学习过程的笔记(写给自己,粗糙也发);
   博客放写给读者的成文文章(有头有尾)。同一个题材可以先在这里长,
   熟了再写成博客文章——**学习记录是苗圃,博客是花坛**;
2. **vs 设计整合 —— 成果归整合,过程归这里**:学设计时蒸馏出的可复用认知
   (规则/数值/方法)进设计整合;学的过程、弯路、练习记录进学习记录。
   同一段学习可以两头各产一篇,各归各位(和素材库"物件/认知"那刀同构);
3. **vs 项目记录 —— 看主角**:一篇的主角是"懂了什么"归学习记录;
   主角是"项目做到哪了"归项目记录。跟着教程做的练习,知识心得写这边、
   项目过程写那边,互不重复。

三个文字板块至此形状各异:项目记录围着**项目**转、学习记录围着**学习线**转、
(代码库待定义)——非同构,验证了当初分三个集合的决定。

## 二、基本盘(已拍板)

- **写作地**:Obsidian 发布文件夹的本板块子目录(对接 spec 第二节);
- **学习来源**:书、免费教程/官方文档/视频、动手实践(做中学)。
  **没有付费课程**——版权红线因此写得轻(见第四节),但预留了一条;
- **双语**:像设计整合那样可选(对接 spec 已定)——默认中文,想双语的篇目
  在 `en/` 写同名英文版;
- **导航名字**:中文站「**学习记录**」,英文站「**Learning Log**」
  (Log 强调轨迹感,也避开 Design Notes 的 Notes);
- **隐藏开关同项目记录**:不想公开的笔记不放进发布文件夹,留在 Obsidian
  只有自己能看(公开仓库+静态站不存在"在线但仅自己可见",机制见项目记录
  spec 第四节,不重复造)。

## 三、内容模型:一篇 = 一个 md 文件,学习线归组

学习和项目不同:**长线**(学全栈可能学一年,没有"完结"一说)、**粒度小**
(今天懂一个概念就可以记一笔)。所以归组单位不是"项目"而是「**学习线**」——
一条持续的学习脉络,如「全栈开发」「字体排印」,一本书也可以是一条线。

字段(基线三件套之外,本板块特有的):

| 字段 | 说明 |
|------|------|
| 标题 + 日期 + 一句话简介 | 基线三件套(对接 spec 第四节,缺一样同步时拦下) |
| **学习线** | **必填,归组键**——同一条线的笔记填同一个名字,页面靠它聚合 |
| 来源 | 选填——这篇学的东西从哪来(书名、教程/文档链接),读者能顺藤摸瓜;纯自己踩坑的不填 |

- 学习线不需要单独建档案文件,线页由归组键自动聚合(同项目记录的做法);
- 学习线名字一旦定了别改(归组键,改名等于拆线;真要改就整线一起改)。

## 四、来源与红线:引用规矩(比项目记录轻,但立住)

学习笔记天然要引用别人的东西(书、教程、文档),规矩三条:

1. **自己的话重述为主**:笔记的价值是你的理解,不是复读。短引可以
   (一两句、为了讨论它),**整段照抄不行**——那是转载不是笔记;
2. **凡引用标出处**:书名/作者、教程/文档的链接——来源字段之外,
   正文里引到哪儿标到哪儿;
3. **不贴扫描和截图**:书页拍照/扫描、视频课截屏,都是把别人的排版和画面
   再分发,不进公开仓库。图想要就自己画/自己截自己的屏幕。

**预留一条**(现在用不上,将来有了付费课程再激活):付费课程的课件、图表、
课程代码一律不贴,自己的话重述才能发。

配图照旧走管线(对接 spec 的自动关卡);学习笔记截自己的屏幕时,
注意画面里别带无关窗口和个人信息(管线抹的是元数据,画面自己把关)。

## 五、页面结构:学习线为纲 + 全部时间流入口

| 页面 | 内容 | 定稿程度 |
|------|------|----------|
| 板块首页 | 学习线列表(每线一张卡:线名 + 篇数 + 最近更新) | 只定底线 |
| 学习线页 | 该线全部笔记按时间排——一条线的成长轨迹 | 只定底线 |
| 文章页 | 普通文章版式,和全站统一;有来源的显示来源 | 结构已定 |
| 时间流入口 | 全板块按时间倒序的"最近在学什么"视图 | 只定底线 |

具体版式交给视觉设计,这里只定"按线能聚合、轨迹能顺着看、有一个时间流视图"的底线。

## 六、写作与发布流(发一篇的完整动作)

1. 在 Obsidian 该板块目录随手记(粒度小是特性,一个概念就值一篇);
2. 填字段:基线三件套 + 学习线(+ 有来源填来源);
3. 发布前自查第四节引用规矩(重述为主、出处齐、无扫描截图);
4. 移进发布文件夹 → 私有笔记库按同步按钮(对接 spec 管线接手);
5. 网站要更新时照旧手动部署(同步 ≠ 上线)。

## 七、明确不做(YAGNI)

- 不做阶段标记(在学/搁置/学完)——学习很少真正"学完",最近更新日期就够了;
- 不做来源打分/书评推荐度——那是博客书评的领地;
- 不做标签、搜索、评论;学习线归组够用;
- 不做进度条、打卡日历、连续天数这类量化装饰;
- 不做"苗圃自动长成花坛"的机制——笔记毕业成博客文章是人工改写,不是搬运。

## 八、开工条件

`2026-07-18-obsidian-sync-spec.md` 建成跑通;③ 用户点名开工。

# 个人网站 — 设计方案(2026-07-16 用户已确认)

## 定位

设计师的个人网站:**作品集为主、博客为辅**,中英双语,免费托管、公开访问。
维护方式:近两个月由 Claude 对话更新内容;之后用户会换电脑、换 Claude 账号,
整个项目必须可平滑迁移(详见 AGENTS.md「换电脑/换 Claude 账号」一节)。

## 技术选型

**Astro 静态站**(用户在三个选项中确认):
- 每篇文章/每个作品是一个独立 markdown 文档,站点自动生成列表、归档、双语切换
- 构建产物是纯静态网页,免费托管、迁移都最省事
- 备选项(纯手工 HTML、Next.js)因维护成本和迁移风险被排除

## 页面结构

| 页面 | 内容 |
|------|------|
| 首页 | 自我介绍 + 精选作品,访客 10 秒内知道站主是干什么的 |
| 作品集 | 列表页 + 每个作品的详情页(大图 + 说明) |
| 博客 | 文章列表 + 文章页 |
| 关于/联系 | 自我介绍与联系方式 |

- 每页都有中英两个版本,URL 形如 `/zh/…` 与 `/en/…`,页面上一键切换
- 内容即文件:作品与文章放内容集合(content collections),双语内容两个语言版本都要有

## 视觉方向(暂定,后期可能整体改向)

- 沿用 Creative OS 设计语言:卡纸质感、craft.do 式克制感
- 参考材料:`reference/creative-os-design/`(只读快照,**只复制不引用**)
- 视觉探索与改向由「设计辅助对话」负责产出选项,用户拍板
- 字体授权约束见 AGENTS.md 设计约束一节(随站分发必须附许可正文;RFN 规则逐个核对)

## 上线方式

- GitHub Pages,免费,推送后自动部署(部署配置在实现阶段做)
- 已知取舍:GitHub Pages 在国内访问时好时坏;若成为问题,后续加自定义域名改善
- 远程仓库尚未建立;首次推送前需用户明确确认

## 明确不做(YAGNI)

- 不做后台管理界面/CMS
- 不做评论、账号、会员等任何需要服务器的功能
- 不引入 React 等前端框架,静态输出够用

# 视频/音频播客板块 — 需求文档(2026-07-17 用户澄清确认)

> 九大板块之四。三轮需求澄清的落字结果,**只定义、不搭建**:
> 页面代码要等视觉方向定稿(`tokens.css`)且用户点名开工才写(宪法第八条,先规格后代码)。
> 内容本身**还没开始做**,所以这份文档刻意写得轻:定骨架和边界,不预设细节。
> 体例与决策方式参照 `2026-07-17-photo-gallery-spec.md`。

## 一、定位:它和已定义板块的本质区别

作品集和影集的内容本体(图、文)**都在仓库里**;这个板块的内容本体
(视频/音频文件)**永远在站外平台**——文件太大,仓库和免费托管都装不下,
这是物理约束不是偏好。

网站上的一个条目只是内容的**「壳」**:播放器(或跳转链接)+ shownotes 文字。
**能不能看,取决于外部平台活不活**——这是它独有的风险形状,
作品集/影集/博客都没有这个依赖。所以不能套任何已有模板。

## 二、基本盘(已拍板)

- **内容现状**:还没开始做,属于未来计划。板块先定义好,等内容。
- **视频和音频是一回事**:混在一个列表里按时间排,不分家。
- **呈现方式:能嵌就嵌、不能就跳**——支持嵌入的平台在页面里直接播,
  不支持的放跳转链接。具体哪个平台怎么处理,等真有内容、真选了平台再定。
- **系列:先单条平铺**。所有条目一个列表,不分节目/系列;
  将来真有固定名字的连续节目,再加一个"属于哪个系列"的归组(改动很小,现在不做)。
- **管理地**:shownotes 直接在网站仓库写(每条一个 md 文件),**不经过 Obsidian**——
  文字量也就每条一段,理由同影集(为少量文字不值得做对接)。
- **导航名字**:中文站「**播客**」,英文站「**Podcast**」——视频也装在里面。

## 三、内容模型:一条 = 一个 md 文件

字段(用户确认真会填的):

| 字段 | 说明 |
|------|------|
| 标题 + 日期 | 列表排序和识别的最低配置 |
| 一句话简介 | 列表里每条下面那行字 |
| 封面图 | 每条一张,入库前过 `docs/图片入库工序.md` 的管线 |
| 平台链接 | 内容本体所在处;嵌入播放器或跳转按钮都从它来 |
| 正文 = 长简介/时间轴 | shownotes:本期讲什么、分段时间点,详情页展示 |

- **双语(已拍板)**:中英各一份——每条建条目时标题/简介/shownotes 两种语言都写,两站都全。
- 封面之外,条目不含任何媒体文件(见第一节,文件本体在平台)。

## 四、页面结构

| 页面 | 内容 | 定稿程度 |
|------|------|----------|
| 播客列表页 | 封面 + 标题 + 日期 + 一句话简介,按时间排 | 结构已定 |
| 条目详情页 | 播放器(能嵌)或醒目跳转按钮(不能嵌)+ shownotes | 结构已定 |

具体版式、播放器样式交给视觉设计,这里只定"列表 → 详情、详情里能播或能跳"的骨架。

## 五、入库工作流(发一期的完整动作)

1. 把视频/音频发布到外部平台(B站/YouTube/小宇宙等,平台自选);
2. 拿到这一期的平台链接;
3. 建条目 md:填标题、日期、一句话简介、平台链接,写 shownotes(中英各一份);
4. 封面图过 `scripts/prep-image.mjs` 后放入;
5. 本地预览确认 → `npm run check` + `npm run build` 双关 → 提交。

## 六、隐私与安全红线(仓库和网站都公开)

- shownotes 别写不打算公开的信息(合作方、未公开项目等),它和博客一样全网可见;
- 封面图一律过入库脚本(顺带抹掉 EXIF/GPS,红线注释在脚本里);
- **嵌入播放器 = 页面里引入平台的代码**:访客的浏览器会直接连那个平台。
  两个后果要心里有数:① 国内访客打不开 YouTube 嵌入(选平台时记得);
  ② 访客的访问记录会被那个平台看到。不想引入第三方就用跳转链接;
- 平台把视频删了/账号没了,条目就成空壳——链接有效性在上线前检查清单里过一遍。

## 七、明确不做(YAGNI)

- 不在仓库里放任何视频/音频文件(物理装不下,也不该装);
- 不做站内播放列表、RSS 订阅源(平台自带,不重复造);
- 不做评论、弹幕、播放量统计;
- 不做系列/节目归组(暂,见第二节);
- 不做 Obsidian 对接(理由见第二节)。

## 八、开工条件

同时满足才动手:① 视觉方向定稿(`tokens.css` 就位);② 用户点名开工——
考虑到内容还没开始做,这个板块大概率排在有内容的板块之后。
届时实现细节(嵌入怎么做、各平台怎么判断)另出实现计划,仍先给用户过目。

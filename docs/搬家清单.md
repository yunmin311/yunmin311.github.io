# 搬家清单 — 换电脑时照着做(人话版)

> 前提:网站已经推送到 GitHub 云端。如果还没推送过,先在旧电脑上让 Claude 完成首次推送,
> 否则云上没有备份,搬不了。

## 新电脑上要做的事

1. **装两个软件**(都免费,官网下载,一路下一步):
   - Node.js —— https://nodejs.org (选 LTS 版本)
   - Git —— https://git-scm.com

2. **把网站取回来**:装好 Claude Code 后,新开一个对话,对它说:
   > 「把我 GitHub 上的个人网站仓库取到 D:\personal-site,然后读一遍里面的 AGENTS.md,
   > 按里面的规矩接手这个项目。」
   它会引导你完成 GitHub 登录授权(在浏览器里点确认就行,不需要你懂命令)。

3. **让它自检**:对它说:
   > 「装好依赖,跑一遍构建,确认网站在新电脑上能正常工作。」
   它跑通后会告诉你结果。

## 换 Claude 账号

**什么都不用做。** 这个项目不依赖任何 Claude 账号里的东西——
新账号的新对话只要读了项目里的 AGENTS.md,就知道该怎么干活。

## 旧电脑退役前的检查(让旧电脑上的 Claude 做)

- 「看看有没有还没提交、还没推送的改动,有就全部提交并推送」
- 确认它回答"工作区干净、远程已是最新"再关机

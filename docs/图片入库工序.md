# 图片入库工序(人话版)

> 场景:你有一张要放上网站的图——手机照片、扫描件、导出的大图。
> 原图往往好几 MB,直接进仓库会把仓库撑肥、网页变慢。照下面三步走。

## 第一步:把原图变成"网页尺寸"

在项目文件夹打开终端,跑:

```bash
node scripts/prep-image.mjs "原图的完整路径"
```

它会在原图旁边生成一个 `原名-web.jpg`:最长边缩到 2400 像素、体积大幅变小,
跑完会告诉你压缩前后的体积。想指定输出位置就在后面多写一个路径。

**iPhone 照片是 HEIC 格式怎么办?** 这个脚本不认 HEIC。两个办法:
- 电脑上用 Windows「照片」打开它 → 另存为 JPG,再跑脚本;
- 或者一劳永逸:iPhone 设置 → 相机 → 格式 → 选「兼容性最佳」,以后拍的直接就是 JPG。

## 第二步:把压好的图放进内容文件夹

- **作品正文里的图**:放 `src/content/works/_images/`
- **作品封面**:放 `src/content/works/_covers/`
- **博客文章里的图**:放 `src/content/posts/_images/`(第一次用时新建这个文件夹即可)

文件名用英文小写加连字符,例如 `poster-final-2026.jpg`(中文名容易在别的电脑上出乱码)。

## 第三步:在文章里引用

markdown 正文里写(注意 `../` 开头,从文章文件出发去找图):

```markdown
![这张图的文字说明](../_images/poster-final-2026.jpg)
```

封面则在 frontmatter 里写:`cover: ../_covers/xxx.jpg`。

## 之后发生什么(你不用管,但可以知道)

构建时 Astro 会自动把每张图再压成多档尺寸的 webp(手机拿小图、大屏拿大图),
浏览者按需下载。这是全自动的——你只要保证入库的图别超过 2400px、别是 HEIC 就行。
一句话:**入库前跑一遍脚本,剩下交给构建。**
